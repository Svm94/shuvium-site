## 2025-04-08 - Custom UI Toggle Group Accessibility
**Learning:** Custom UI toggle groups lacking native radio semantics require explicit structural and state annotations to be accessible. Specifically, they must be wrapped in a container with `role="group"` and an `aria-label`, interactive elements must use `aria-pressed` to indicate their current state, and `focus-visible` utility classes should be included to support keyboard navigation explicitly.
**Action:** When creating custom toggle groups or segmented controls, always apply `role="group"`, an appropriate `aria-label`, `aria-pressed` for active states, and `focus-visible` classes for keyboard users.
## 2025-05-18 - Accessibility Bypass Blocks
**Learning:** Adding global app-level "Skip to content" bypass blocks requires careful routing design. It's best implemented by creating a global layout wrapper (`<RootLayout />` using React Router's `<Outlet />`) rather than modifying individual pages, ensuring it applies universally and early in the DOM. Furthermore, using CSS transforms (`-translate-y-[200%]`) instead of positional values (like `top`) creates smoother visual transitions for users when the element gains `focus-visible`.
**Action:** Always implement a visually hidden "Skip to content" link in applications to enhance keyboard accessibility. Ensure the target main content container has a matching ID, `tabIndex={-1}`, and `focus-visible` styling to clearly indicate focus transfer.
