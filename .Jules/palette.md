## 2025-04-08 - Custom UI Toggle Group Accessibility
**Learning:** Custom UI toggle groups lacking native radio semantics require explicit structural and state annotations to be accessible. Specifically, they must be wrapped in a container with `role="group"` and an `aria-label`, interactive elements must use `aria-pressed` to indicate their current state, and `focus-visible` utility classes should be included to support keyboard navigation explicitly.
**Action:** When creating custom toggle groups or segmented controls, always apply `role="group"`, an appropriate `aria-label`, `aria-pressed` for active states, and `focus-visible` classes for keyboard users.
## 2025-05-04 - Skip-to-content focus validation
**Learning:** When using Playwright TS/JS to verify visually hidden elements that use Tailwind arbitrary value classes containing brackets (e.g., `-translate-y-[200%]`), the assertion `toHaveClass` requires proper escaping in regular expressions (e.g., `/.*-translate-y-\[200%\].*/`) to correctly match the class string. Additionally, for testing the focus behavior of visually hidden elements natively, `.focus()` followed by a `page.waitForTimeout()` correctly triggers standard `focus-visible` UI transitions without needing simulated keyboard events.
**Action:** When verifying element visibility transitions dependent on focus states in Playwright, prefer direct `locator.focus()` with a slight timeout, and always escape square brackets in class-matching regular expressions for Tailwind arbitrary values.
