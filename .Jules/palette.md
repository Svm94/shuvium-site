## 2025-04-08 - Custom UI Toggle Group Accessibility
**Learning:** Custom UI toggle groups lacking native radio semantics require explicit structural and state annotations to be accessible. Specifically, they must be wrapped in a container with `role="group"` and an `aria-label`, interactive elements must use `aria-pressed` to indicate their current state, and `focus-visible` utility classes should be included to support keyboard navigation explicitly.
**Action:** When creating custom toggle groups or segmented controls, always apply `role="group"`, an appropriate `aria-label`, `aria-pressed` for active states, and `focus-visible` classes for keyboard users.

## 2025-04-28 - Skip to Content Implementation Standard
**Learning:** Adding a "Skip to content" link requires coordinated changes across routing layouts and specific CSS constraints. The link must be focusable via `tabIndex` behavior, but its target (usually a `<main>` container) must have `tabIndex={-1}` and `focus-visible:outline-none` so it properly catches the programmatic focus shift without interrupting the keyboard tab order or displaying an unsightly focus ring on the entire page wrapper. Furthermore, negative CSS translations (`-translate-y-[200%]`) provide a significantly smoother visual transition upon focus than dynamically altering position properties (e.g., `top:-100px`).
**Action:** Always implement bypass blocks as fixed, negative-translated links early in the layout hierarchy (e.g., `Layout.tsx`), and verify the target container has `tabIndex={-1}` and appropriate `focus-visible` styles to handle programmatic focus elegantly.
